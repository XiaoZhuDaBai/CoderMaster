# 【纯 solutionCode 生成提示词】

> 只生成一个 Java 正解代码，必须经过样例验证后才能输出。
> 不生成 generatorCode，不生成 expectedOutput，不生成任何 JSON。

---

## 【输出格式】

整个回复只能包含一个 JSON 对象，首字符为 `{`、尾字符为 `}`，禁止任何 Markdown 标记、解释文字或多余文本。

JSON 字段：
- `solutionCode`：String，经过样例验证后的 Java 正解代码（完整可编译运行）

---

## 【第一步：理解题目】

仔细阅读下面的 problem_description：
- 题目要求解什么问题（算法本质）
- 输入格式是什么（inputDesc）：有哪些行、每行什么含义、有没有多组测试数据
- 输出格式是什么（outputDesc）
- 约束条件是什么（极值、数据规模）

---

## 【第二步：写出 solutionCode】

- **类名必须是 `public class Main`**，禁止使用 `class Main`（必须带 public 修饰符），禁止有任何 package 声明
- 使用 `Scanner` / `BufferedReader` 读取标准输入
- **必须严格按题目 inputDesc 的格式解析输入**：
  - 如果 inputDesc 说第一行是 T 表示测试组数，就必须先读 T，再循环读 T 组数据
  - 如果 inputDesc 说第二行是 n 和 K，就必须一次读两个整数
  - **绝对不能凭自己的猜测改变输入格式**
- 严格按题目 outputDesc 输出结果
- 可使用任何 JDK 17 标准库

---

## 【第三步：用沙箱执行思维链验证 solutionCode】

> **这一步是本提示词最核心的环节。跳过此步意味着题目可能被完全理解错。**

**必须严格模拟沙箱执行流程来验证：**

1. **模拟编译**：检查 solutionCode 是否有语法错误（缺少分号、括号不匹配等）
2. **模拟执行**：对 sample_cases 中的每个 input，完全模拟代码执行路径：
   - 模拟 Scanner/BufferedReader 逐行读取 input
   - 模拟每一步变量变化（不是"猜答案"，而是严格跟踪执行流程）
   - 记录实际输出结果
3. 对比模拟输出与 expectedOutput

**如果两者不一致**：
- 说明 solutionCode 的算法逻辑或输入解析有误
- 必须回到第二步重新分析题意并修正代码
- 重复此验证步骤，直到所有样例都通过

**【关于"选择"类题目的额外验证】**：
- 如果题目涉及"选择最优方案"，必须枚举所有可能的方案
- 计算每种方案的结果，对比确认哪个对应 expectedOutput
- 如果所有方案都不匹配 expectedOutput，说明题目描述可能有歧义，尝试换一种理解方式

**禁止**：
- 禁止仅凭感觉猜测输出
- 禁止跳过验证直接输出 solutionCode
- 禁止在验证不通过时仍输出 JSON

**【重要】关于"选择"类题目的验证**：
- 如果题目涉及"选择是否"、"选择跳过"等操作，必须枚举所有可能的方案
- 在手算时必须明确列出：
  1. 方案A：选择哪些天/步骤（跳过哪些）
  2. 方案B：选择哪些天/步骤（跳过哪些）
  3. 计算每种方案的结果
- 比较不同方案的结果，确认哪种是最优解
- 如果你的解法与 expectedOutput 不符，说明：
  (a) 你的解法有问题，需要修正
  (b) expectedOutput 有问题（较少见，但可能出现）

---

## 【第四步：通用验证规则】

> **适用于所有题目类型，不仅是"选择"类题目**

**(A) 枚举验证法** - 列出所有可能的解/路径，对比得出最优：
- 不要只验证一种解法就认为正确
- 必须列出所有合理的解法/选择路径
- 计算每种解法的结果
- 确认 expectedOutput 对应哪种解法

**(B) 边界验证法** - 检查边界条件和特殊情况：
- 空输入 / 零值输入的结果是什么？
- 只有一个元素/步骤时的结果？
- 最大/最小输入时的可行性？

**(C) 交叉验证法** - 用不同思路验证同一结果：
- 用公式验证 vs 用枚举验证
- 用贪心验证 vs 用动态规划验证

**【强制要求】**：
- 必须枚举至少 2 种解法进行比较（如果存在多种解法）
- 只有当你的解法与 expectedOutput 完全一致时，才认为通过
- 如果不一致，必须重新分析题目或修正代码
- **禁止跳过验证直接输出 solutionCode**
